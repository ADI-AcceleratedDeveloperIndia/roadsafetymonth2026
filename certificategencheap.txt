1. TECHNICAL APPROACH
================================================================================

1.1 CURRENT IMPLEMENTATION
--------------------------
Method: Browser-Based PDF Generation (Client-Side)
Technologies: html2canvas + jsPDF (JavaScript libraries)
Hosting: Vercel Pro Plan with Global CDN

Architecture Flow:
1. User opens website on phone/laptop
2. User fills certificate form (Name, District, Type)
3. User clicks "Preview Certificate"
4. Certificate renders as HTML/CSS in browser
5. User clicks "Download PDF"
6. Browser downloads html2canvas.js (400 KB) - cached after first load
7. Browser downloads jspdf.js (200 KB) - cached after first load
8. Browser converts certificate HTML → Canvas → PNG → PDF
9. PDF saves directly to user's device

Server Role: Only serves static files (HTML, JS, CSS, images)
PDF Generation: 100% in user's browser (phone/laptop CPU)

Key File Locations:
- PDF Export Logic: utils/certificateExport.ts
- Certificate Component: components/certificates/Certificate.tsx
- Preview Page: app/certificates/preview/page.tsx
- Generate Form: app/certificates/generate/page.tsx

1.2 WHY BROWSER-BASED vs SERVER-BASED?
---------------------------------------

BROWSER-BASED (Current - RECOMMENDED):
✓ Zero server CPU usage
✓ Infinite scalability
✓ No timeout issues
✓ Fast user experience (2-3 seconds)
✓ Cost-effective (only bandwidth charges)
✓ Each user's device does the work

SERVER-BASED (Puppeteer - NOT RECOMMENDED):
✗ High server CPU usage (95-100%)
✗ Limited to ~500 concurrent users max
✗ 10-second timeout risks on Vercel
✗ Expensive (₹50+ lakhs per month)
✗ Cold start delays (10-15 seconds)
✗ Would CRASH with 5 lakh concurrent users

Note: A server-side API exists (app/api/certificates/download/route.ts) for
admin bulk exports and email attachments only - NOT for public downloads.

3.7 COST OPTIMIZATION RECOMMENDATIONS
--------------------------------------

To reduce costs further:

1. Image Optimization
   - Convert PNG logos to WebP format (50% smaller)
   - Compress photos to 80% quality (visually identical)
   - Expected savings: 40% bandwidth = ₹1.2 lakhs saved

2. Aggressive Caching
   - Set cache headers: Cache-Control: public, max-age=31536000
   - Libraries cached for 1 year in user's browser
   - Expected savings: 30% bandwidth = ₹90,000 saved

3. Lazy Loading
   - Load certificate images only when preview is opened
   - Reduces unnecessary bandwidth for users who abandon form
   - Expected savings: 15% bandwidth = ₹47,500 saved

4. Progressive Web App (PWA)
   - Cache entire application in user's browser
   - Works offline after first visit
   - Expected savings: 50% for return visitors = ₹1.5 lakhs saved

Total Potential Savings: ₹2.85 lakhs
Optimized Peak Month Cost: ₹32,060 (down from ₹3.17 lakhs)
